UDP Message

You can use User Datagram Protocol (UDP) for one-to-many communication when low latency is more important than lossless data and you do not require guaranteed delivery. This sample project demonstrates how to use UDP to broadcast messages on demand from the host computer and to receive the messages on the myRIO.

This sample project includes the following features:

  • Message-based communication model—Refers to the infrequent information transfer that a specific event triggers. The specific event can be a button press, an alarm, and so on. In this model, the receiver must process the message before receiving the next message. This model delivers messages with a low latency. Message-based communication is also known as command-based communication.
  • UDP—Provides simple and low-level communication among processes on multiple computers. Processes communicate by sending datagrams to a destination computer or port. A port is the location where you send data. After the datagram reaches the destination computer, UDP moves the datagram to its destination port. If the destination port is not open, UDP discards the datagram. UDP is not a connection-based protocol, so you do not need to establish a connection with a destination before you send or receive data. Instead, you specify the destination for the data when you send each datagram.
  • UDP VIs and functions—Open and close a UDP connection capable of reading and writing UDP data.
  • LED Express VI—Sets the state, either ON or OFF, of the LEDs on the myRIO.

Understanding the Components of This Sample Project

This sample project consists of the following VIs:

  • Desktop Main.vi—Opens a UDP network connection on the specified port of the host computer, writes the LED states to the network connection, and closes the network connection. This VI runs on the desktop computer. This VI uses an Event structure to handle the following events:
    • LED—When you change the value of LED on the Desktop Main VI, this VI sends the updated value to the network connection.
    • Stop—When you click Stop on the Desktop Main VI, this VI stops.
  • RT Main.vi—Opens a UDP network connection on the specified port of the myRIO target, reads the LED states from the network connection, sets the states of the LEDs on the myRIO to match the LED states, and closes the network connection. This VI runs on the myRIO.

Running This Sample Project

Complete the following steps to run this sample project:

Note  Before you run this sample project, make sure you connect the myRIO to the host computer.

  1. Open and run the Desktop Main VI under My Computer.
  2. Open and run the RT Main VI under your myRIO target.
  3. Change the value of LED on the Desktop Main VI.
  4. Observe the LEDs on the myRIO and the LED Status on the RT Main VI. The LED states and the value of LED Status match the value of LED on the Desktop Main VI.
  5. Click Stop on both the Desktop Main VI and the RT Main VI to stop the application. The Desktop Main VI and the RT Main VI run independently of each other.
